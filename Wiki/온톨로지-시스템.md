# 온톨로지 시스템 (Ontology System)

> Kcode v18.5.0+ 에서 사용 가능

## 1. 개요

Kcode의 온톨로지 시스템(K엔진)은 코드 안에 지식 그래프를 직접 내장하는 기능입니다.  
개발자는 **"무엇을 알고 있는지"** 만 정의하면, 추론·검증·쿼리는 엔진이 자동 처리합니다.

### 3가지 동작 모드

| 모드 | 설명 | 대상 |
|------|------|------|
| 내장 모드 | K엔진이 언어 런타임에 통합 | `.han` 소스에서 직접 사용 |
| 서버 모드 | `kont_server`로 독립 실행 | REST / WebSocket API |
| 대여 모드 | `libkc_ontology.so` | Python / Node.js / C 앱 |

## 2. 개념(Concept) 정의

```han
온톨로지 "스마트공장":

    // 개념 정의
    개념 장비
    개념 기계 상위=장비 속성=[이름:문자, 온도:실수, 가동중:논리]
    개념 제품 속성=[ID:문자, 품질등급:문자]
    개념 불량품 상위=제품 속성=[원인:문자]

    // 관계 정의
    관계 기계 → 생산 → 제품
    관계 기계 → 감지 → 불량품

    // 제약 조건 (엔진이 자동 검증)
    제약 온도 < 150.0
    제약 온도 >= -10.0

온톨로지끝
```

## 3. 지식 데이터 불러오기

4가지 포맷을 지원합니다.

### K문법 (`.han`) — 가장 간단

```han
온톨로지불러오기("factory_data.han")
```

### JSON-LD

```json
{
  "@context": { "기계": "http://kcode.io/factory#기계" },
  "@graph": [
    { "@id": "기계:A001", "@type": "기계", "온도": 72.5, "상태": "가동중" }
  ]
}
```

```han
온톨로지불러오기("factory_data.jsonld", 형식=JSON-LD)
```

### CSV — 대량 인스턴스 입력

```csv
ID,유형,온도,상태
A001,기계,72.5,가동중
A002,기계,65.0,대기
```

```han
온톨로지불러오기("machines.csv", 형식=CSV, 유형열="유형")
```

### Turtle (`.ttl`)

```turtle
@prefix kc: <http://kcode.io/factory#> .
kc:기계A001 a kc:기계 ; kc:온도 72.5 ; kc:상태 "가동중" .
```

```han
온톨로지불러오기("relations.ttl", 형식=Turtle)
```

## 4. 런타임 인스턴스 추가

파일 없이 실행 중에 직접 추가합니다.

```han
온톨로지삽입 개념=기계 인스턴스:
    이름 = "B003"
    온도 = 88.3
    가동중 = 참
온톨로지삽입끝
```

## 5. 온톨로지 쿼리

### 한글 쿼리

```han
// 기본 검색
결과 = 온톨로지검색("기계 찾기")
각각 항목 안에 결과:
    출력(항목.이름 + " / " + 항목.온도)

// 조건 검색
위험기계 = 온톨로지검색("기계 중 온도 > 100")

// 관계 탐색
제품목록 = 온톨로지검색("기계A001이 생산한 것")
```

### SPARQL 쿼리 (고급)

```han
쿼리 = """
    SELECT ?기계 ?온도
    WHERE {
        ?기계 a kc:기계 .
        ?기계 kc:온도 ?온도 .
        FILTER (?온도 > 80.0)
    }
"""
결과 = 온톨로지SPARQL(쿼리)
```

## 6. kont_server — 온톨로지 독립 서버

```bash
# 빌드
gcc -O2 ... kc_ont_server.c -o kont_server

# 실행
./kont_server                   # 기본 포트
./kont_server 9090              # 포트 지정
```

### REST API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `POST` | `/ont/class` | 클래스 추가 |
| `POST` | `/ont/instance` | 인스턴스 추가 |
| `GET` | `/ont/query` | 한글 쿼리 실행 |
| `POST` | `/ont/sparql` | SPARQL 쿼리 실행 |
| `GET` | `/ont/export` | 전체 데이터 내보내기 |
| `GET` | `/health` | 서버 상태 확인 |

## 7. 제약 조건과 계약 연동

온톨로지 제약 조건은 Kcode 계약 시스템과 자동 연동됩니다.

```han
온톨로지 "안전규정":
    개념 기계 속성=[온도:실수, 압력:실수]
    제약 온도 < 150.0      // 위반 시 자동으로 법위반 처리
    제약 압력 < 10.0
온톨로지끝

// 실행 중 자동 검증
온톨로지삽입 개념=기계 인스턴스:
    온도 = 200.0    // ← 제약 위반 → 엔진이 자동 감지
온톨로지삽입끝
```
