# 컴파일러 구조 (Architecture)

## 1. 전체 파이프라인

```
.han 소스 파일
      │
      ▼
  [klexer.c]
  렉서 (Lexer)
  소스 텍스트 → 토큰 스트림
      │
      ▼
  [kparser.c]
  파서 (Parser)
  토큰 스트림 → AST (추상 구문 트리)
      │
      ├─────────────────────────────────────┐
      ▼                                     ▼
[kc_bcgen.c]                        [kcodegen.c / kcodegen_llvm.c]
바이트코드 컴파일러                    코드 생성기
AST → .kbc 바이트코드                 AST → C코드 또는 LLVM IR
      │                                     │
      ▼                                     ▼
 [kc_vm.c]                          gcc / clang 빌드
 KVM 가상머신                              │
 바이트코드 실행                           ▼
                                      네이티브 실행파일
```

## 2. 소스 파일 구성

| `kc_mcp.h` / `kc_mcp.c` | MCP 서버 런타임 |

### 레이어 2 — 실행 백엔드

| `kc_vm.h` / `kc_vm.c` | KVM 스택 기반 가상머신 |

### 레이어 3 — 코드 생성기

## 3. 4가지 실행 모드

### 모드 1 — 바이트코드 VM (권장)

```
소스.han → [klexer] → [kparser] → [kc_bcgen] → .kbc → [kc_vm] → 실행
```

- 가장 빠른 개발 사이클
- 소규모 스크립트, 교육, 빠른 테스트에 적합

### 모드 2 — C 코드 생성

```
소스.han → [렉서+파서] → [kcodegen] → 소스.c → [gcc] → 실행파일
```

- 외부 의존성 없는 배포 바이너리 생성
- 이식성이 가장 높음

### 모드 3 — LLVM IR

```
소스.han → [렉서+파서] → [kcodegen_llvm] → 소스.ll → [clang] → 실행파일
```

- 최적화 수준이 가장 높음
- 다양한 타겟 아키텍처 지원 (ARM, x86-64, RISC-V 등)

### 모드 4 — WASM (브라우저)

```
소스.han → [렉서+파서] → [kc_bcgen] → WASM → 브라우저에서 실행
```

- kserver 없이 웹 브라우저에서 직접 실행
- 온라인 데모, 교육 환경에 적합

## 4. GC (가비지 컬렉터)

Kcode는 2단계 GC를 사용합니다.

```
단계 1 — 참조카운트 (Reference Counting)
  → 즉시 해제 가능한 객체를 빠르게 정리

단계 2 — 마크스윕 (Mark & Sweep)
  → 순환 참조를 포함한 나머지 객체 정리
  → 주기적으로 실행
```

개발자는 `해제()` 함수로 명시적 해제도 가능합니다.

## 5. 텐서 + 자동미분 엔진

AI/머신러닝 용도를 위한 자동미분 내장 함수입니다.

```han
텐서 x = 텐서([2.0], [1])
텐서 w = 텐서([3.0], [1])
미분추적(x)
미분추적(w)

텐서 y = 미분더하기(미분곱(x, x), 미분곱(w, 텐서([2.0],[1])))
역전파(y)

출력(x.기울기)   // dy/dx = 2x = 4.0
출력(w.기울기)   // dy/dw = 2.0
```

| 내장 함수 | 설명 |
|-----------|------|
| `역전파(t)` | 루트 텐서에서 역전파 실행 |
| `기울기초기화(t)` | grad 버퍼 0 초기화 |
| `미분추적(t)` | requires_grad = 1 설정 |
| `미분더하기(a,b)` | 덧셈 + GradFn |
| `미분곱(a,b)` | 원소별 곱 + GradFn |
| `미분행렬곱(a,b)` | 행렬곱 + GradFn |
| `미분렐루(a)` | ReLU + GradFn |
| `미분시그모이드(a)` | σ(a) + GradFn |

## 6. 셀프 호스팅 로드맵

Kcode는 궁극적으로 Kcode 자신으로 엔진을 재작성하는 것을 목표로 합니다.

```
레벨 1 (현재)   C 엔진으로 .han 실행
레벨 2 (§CLI)   CLI 도구를 .han으로 작성
레벨 3 (§SRV)   서버를 .han으로 재작성
레벨 4 (§WEB)   Web IDE를 .han으로 작성
레벨 5 (§SH)    엔진 자체를 .han으로 재작성 → 언어 완성
```

> **참고**: Go v1.5(2015), Rust(2011), PyPy도 동일한 경로를 걸었습니다.  
> C로 작성된 `klexer.c` / `kparser.c`는 부트스트랩 씨앗으로 영구 보존됩니다.
