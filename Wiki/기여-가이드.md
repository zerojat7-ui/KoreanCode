# 기여 가이드 (Contributing)

## 1. 파일 이름 규칙

모든 소스 파일(`.c`, `.h`) 이름 앞에 `k`를 붙입니다.

```
✅ klexer.c     kparser.c     kcodegen.c
✅ klexer.h     kparser.h     kcodegen.h
❌ lexer.c      parser.c      codegen.c
```

예외:
- `readme.md` — `k` 접두어 없음
- 이미 `k`로 시작하는 파일 — 그대로 유지

---

## 2. 버전 관리 규칙

| 변경 규모 | 버전 올림 | 예시 |
|----------|----------|------|
| 소규모 (파일 1~2개 수정) | 패치 `+0.0.1` | `v1.0.0 → v1.0.1` |
| 중규모 (3개 미만 파일) | 마이너 `+0.1.0` | `v1.0.0 → v1.1.0` |
| 대규모 (3개 이상 파일) | 메이저 `+1.0.0` | `v1.0.0 → v2.0.0` |

- `readme.md` 기준으로 버전을 올립니다
- 각 파트 파일은 `readme.md`의 버전을 상속합니다
- 각 파일 상단 주석에 버전 번호를 기재합니다

## 3. 작업 흐름

### 새 기능 추가 시

1. `readme.md`에 작업 단계를 기록합니다
2. 각 단계를 완료하면 완료 표시(`✅`)를 합니다
3. 문법 검증 후 커밋합니다
4. 완료된 파일은 버전을 올립니다

### 단계 기록 예시 (`readme.md`)

```markdown
### 【v1.2.0】 파일 입출력 추가

| 단계 | 파일 | 작업 내용 | 상태 |
|------|------|----------|------|
| 1 | klexer.h/c | 파일 관련 토큰 17종 추가 | ✅ |
| 2 | kparser.h/c | 파일 노드 파싱 추가 | ✅ |
| 3 | kinterp.c | 파일 내장 함수 17종 구현 | ⬜ |
| 4 | readme.md | 문서 업데이트 | ⬜ |
```

## 4. 코드 스타일

### C 파일

```c
/*
 * klexer.c  —  Kcode 렉서
 * version : v31.0.0
 *
 * 설명을 여기에 씁니다.
 */

#include "klexer.h"
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
```

- 들여쓰기: 공백 4칸
- 함수명: `snake_case` (예: `lexer_init`, `parser_parse`)
- 상수: `UPPER_SNAKE_CASE` (예: `KCODE_MAX_INDENT_DEPTH`)
- 구조체: `PascalCase` (예: `Lexer`, `Parser`, `Node`)

### 문법 검증 필수

코드 작성 후 반드시 아래 명령으로 문법 오류를 확인합니다:

```bash
gcc -fsyntax-only -Wall -std=c11 파일명.c
```

전체 파일 한 번에 검증:

```bash
make test
```

## 5. PR 규칙

### 커밋 메시지 형식

```
[버전] 변경 내용 요약

예시:
[v31.0.1] klexer.c — 파일 확장자 토큰 오타 수정
[v31.1.0] kparser.c kcodegen.c — 열거형 멤버 접근 구문 추가
[v32.0.0] klexer kparser kinterp kcodegen — 다중 반환값 전면 구현
```

### 체크리스트

- [ ] `gcc -fsyntax-only` 검증 통과 (에러 0)
- [ ] `make test` 통과
- [ ] 버전 번호 업데이트 (`readme.md` + 수정된 파일)
- [ ] `readme.md` 작업 단계 완료 표시
- [ ] 새 파일 이름에 `k` 접두어 확인
