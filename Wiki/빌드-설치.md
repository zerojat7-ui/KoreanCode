# 빌드 & 설치 (Build & Install)

## 1. 전체 빌드 순서

```bash
# 1) 소스 클론
git clone https://github.com/zerojat7-ui/KCODE.git
cd KCODE

# 2) 기본 빌드
make

# 3) 시스템 설치 (선택)
sudo make install

# 4) 동작 확인
./kbc exec examples/hello.han
```

---

## 2. Makefile 타겟

| 타겟 | 설명 | 결과 파일 |
|------|------|----------|
| `make` | 전체 빌드 | `kcode_gen`, `kbc` |
| `make gen` | C 코드 생성기 | `kcode_gen` |
| `make kvm` | 바이트코드 VM | `kbc` |
| `make test` | 렉서 단위 테스트 | `ktest_lexer` |
| `make LLVM=1` | LLVM 백엔드 포함 | `kcode_llvm` |
| `make llvm-check` | LLVM 설치 확인 | — |
| `make shared` | 공유 라이브러리 | `libkcode.so` |
| `make static` | 정적 라이브러리 | `libkcode.a` |
| `make install` | 시스템 설치 | `/usr/local/` |
| `make uninstall` | 제거 | — |
| `make clean` | 빌드 결과물 삭제 | — |

---

## 3. CMake 빌드

```bash
mkdir build && cd build
cmake ..
cmake --build .

# LLVM 포함
cmake .. -DKCODE_LLVM=ON
cmake --build .

# WASM 빌드 (Emscripten 필요)
emcmake cmake .. -DBUILD_WASM=ON
cmake --build .
```

---

## 4. LLVM 설치 확인

```bash
make llvm-check
```

LLVM이 없을 때 설치 방법:

```bash
# Ubuntu/Debian
sudo apt install llvm-dev clang

# macOS
brew install llvm
# 이후 PATH 설정 필요:
export PATH="/opt/homebrew/opt/llvm/bin:$PATH"
```

## 5. 공유 라이브러리 빌드 (DLL / SO)

Kcode 엔진을 다른 앱에서 사용할 수 있도록 공유 라이브러리로 빌드합니다.

### Linux / macOS (.so / .dylib)

```bash
make shared
# → libkcode.so.31.0.0 생성
```

### 수동 빌드

```bash
# PIC 오브젝트 컴파일
gcc -fPIC -Wall -std=c11 -c klexer.c  -o klexer.pic.o
gcc -fPIC -Wall -std=c11 -c kparser.c -o kparser.pic.o
gcc -fPIC -Wall -std=c11 -c kinterp.c -o kinterp.pic.o

# 공유 라이브러리 생성
gcc -shared -Wl,-soname,libkcode.so.31 \
    klexer.pic.o kparser.pic.o kinterp.pic.o \
    -lm -o libkcode.so.31.0.0

# 심볼릭 링크
ln -sf libkcode.so.31.0.0 libkcode.so.31
ln -sf libkcode.so.31.0.0 libkcode.so
```

### Windows (MinGW)

```bash
gcc -shared -DKCODE_BUILD_DLL \
    klexer.c kparser.c kinterp.c kcodegen.c \
    -o kcode.dll -Wl,--out-implib,kcode.lib
```

## 6. 시스템 설치

```bash
# 기본 설치 (/usr/local)
sudo make install

# 경로 지정
sudo make install PREFIX=/opt/kcode

# 확인
kbc --version
```

설치 구조:

```
/usr/local/
├── bin/
│   ├── kcode          ← 인터프리터
│   ├── kcode_gen      ← C 코드 생성기
│   ├── kcode_llvm     ← LLVM 생성기 (LLVM=1 빌드 시)
│   ├── kbc            ← 바이트코드 VM
│   └── kserver        ← 로컬 HTTP 서버
├── include/kcode/
│   ├── klexer.h
│   ├── kparser.h
│   ├── kinterp.h
│   ├── kcodegen.h
│   └── kcodegen_llvm.h
└── lib/
    ├── libkcode.so.31.0.0
    ├── libkcode.so.31  → libkcode.so.31.0.0
    └── libkcode.so     → libkcode.so.31.0.0
```

## 7. WASM 빌드

브라우저에서 kserver 없이 Kcode를 실행하기 위한 빌드입니다.

### Emscripten 설치

```bash
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk
./emsdk install latest
./emsdk activate latest
source ./emsdk_env.sh
```

### KVM WASM 빌드 (~200KB)

```bash
emcc -O2 -s WASM=1 \
     -s EXPORTED_FUNCTIONS='["_kc_wasm_run","_kc_wasm_lex","_kc_wasm_parse","_kc_wasm_compile","_kc_wasm_exec_bc"]' \
     -s EXPORTED_RUNTIME_METHODS='["ccall","cwrap","UTF8ToString","stringToUTF8"]' \
     -s ALLOW_MEMORY_GROWTH=1 \
     -DKCODE_WASM_BUILD \
     klexer.c kparser.c kc_bcgen.c kc_vm.c kc_bytecode.c kc_wasm_api.c \
     -o kcode_vm.wasm
```

### CMake WASM 빌드

```bash
emcmake cmake .. -DBUILD_WASM=ON -B build_wasm
cmake --build build_wasm
```

## 8. kserver 단독 빌드

kserver는 외부 의존성 없이 단독으로 빌드 가능합니다.

```bash
# Linux / macOS
gcc -O2 -o kserver kserver.c

# Windows (MinGW)
gcc -O2 -o kserver.exe kserver.c -lws2_32
```

## 9. 정적 vs 공유 라이브러리 비교

| 항목 | 정적 (`.a` / `.lib`) | 공유 (`.so` / `.dll`) |
|------|---------------------|----------------------|
| 파일 크기 | 실행파일에 포함되어 큼 | 실행파일 작고 `.so` 별도 |
| 배포 | 실행파일 하나로 완결 | `.so` 함께 배포 필요 |
| 업데이트 | 재컴파일 필요 | `.so`만 교체 |
| 권장 | 단일 실행파일 배포 | IDE 플러그인, 멀티 도구 |
